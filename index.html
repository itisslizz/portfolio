How I Work

During studies and working on projects I learned one of the most important things when creating something is to keep an oversight. The bigger the project the harder this is. That is why I try to stick to the following plan.

1 - What

The first step is to come up with the "What" i.e. what do I want to build? A simple idea, for example "a photo app" is not enough. I want to think about the concrete functionalities, the platforms and the target group. Once that is figured out, I try to come up with the basic architecture and a first design.

Further down the line, issues may arrise that have not been thought of or initial ideas may not seem like the right choice anymore. This is why these first decisions are not set in stone and might be subject to change.

2 - How

If the "What" is properly defined I move on to step 2, the "How". Here I think about what technologies I want to use for building my project. Which language, which frameworks, which libraries are best suited to help me accomplish my goals.

This is especially hard for web projects, as there is an enormous variety of technologies out there. My thought process also varies. At times I like to stick to what I know in order to feel comfortable and work at a high efficiency from the beginning. Other times I might prefer looking into new or unknown technologies to learn something new and maybe find something I prefer to other things in my repertoire.

3 - Setup

The next step is the "Setup". This includes creating the project, installing the dependencies decided on in step 2, initiating a git repo for version control and get a template ready to expand.

4 - Split It

When working on a bigger project it can quickly occure that one goes in over his head and loses motivation and concentration. To avoid that I split up the project into smaller pieces and tasks (for bigger projects this can also be multi-level). This makes it both easier to keep the oversight as well as reaching goals faster this in turn keeps my motivation high.

5 - Develop

Now looking at the split up project I pick a task, one that is not dependant on other incompleted tasks and try to complete it in two steps and repeat them until I am satisfied.

5.1 - Build It

Write the code and implement or fix the picked task. Keep the code clean, methods small and use comments.

5.2 - Test It

Test the built part of the project, either manually or preferrably write an automated test. If testing fails we go back to fix the part, if it is successful we move on and pick the next task.
